# 주최몇 MVP 규칙

## 목표

사용자가 입력한 손익 금액을 원화 숫자가 아니라 가격대에 맞는 물건과 수량으로 바꿔 보여준다. 결과는 웃기되, 같은 금액대에서 납득 가능한 물건이 나오고 수량 계산이 과장되지 않아야 한다.

## 입력 규칙

- 음수는 손실, 양수는 수익으로 처리한다.
- 빈 값, 숫자가 아닌 값, `0`은 결과를 만들지 않는다.
- 쉼표가 포함된 입력은 숫자로 해석한다. 예: `-3,200,000` -> `-3200000`.
- 결과 계산은 입력값의 절댓값을 기준으로 한다.
- 입력 금액은 클라이언트 상태에서만 사용하며 서버에 저장하지 않는다.

## 가격대 섹션

| 섹션 | 입력 금액 절댓값 | 역할 |
| --- | ---: | --- |
| `under_10k` | 1만원 이하 | 편의점/분식/저가 생필품 |
| `10k_100k` | 1만원 초과 10만원 이하 | 외식/생활 소비/소형 전자 |
| `100k_1m` | 10만원 초과 100만원 이하 | 중형 가전/전자기기/가구 |
| `1m_100m` | 100만원 초과 1억원 이하 | 명품/자동차/고가 경험 |
| `over_100m` | 1억원 초과 | 부동산/초고가 자산/대형 물건 |

## 랜덤 규칙

1. 입력 금액의 절댓값으로 가격대 섹션을 고른다.
2. 선택된 섹션 안의 아이템 중 하나를 랜덤으로 뽑는다.
3. `이미지 다시 뽑기`는 같은 입력 금액을 유지하고 같은 가격대 섹션 안에서 다시 뽑는다.
4. 손실/수익에 따라 배경 팔레트와 하단 문구를 분리한다.
5. 향후 확장 시 같은 섹션 안에서 다음 가중치를 줄 수 있다.
   - 입력 금액과 아이템 가격이 가까울수록 가중치 증가
   - 희귀 아이템은 낮은 확률 유지
   - 같은 아이템 연속 노출 방지

현재 MVP는 코드 기반 가중치 랜덤을 사용한다. 단, 가격대 밖 아이템은 나오지 않는다.

### 등장 확률 규칙

- 같은 가격대 안에서만 아이템을 뽑는다.
- 아이템 가격이 입력 금액보다 비싸면 제외한다.
- 생활형 아이템은 자주 나온다. 예: 라면, 커피, 국밥, 치킨, 팬티
- 고가/초고가 아이템은 낮은 확률로 나온다. 예: 샤넬백, 자동차, 아파트, 요트
- 수량이 너무 커지는 아이템은 확률을 낮춘다. 예: 100만원을 라면 1,000봉지로만 보여주는 상황 방지
- 수량이 `1-10개` 정도로 납득되는 아이템은 확률을 높인다.
- 특정 아이템만 별도 조정이 필요하면 코드의 `weight` 값을 추가해 보정한다.

## 수량 계산 규칙

기본 계산식:

```text
수량 = 입력 금액 절댓값 / 아이템 기준 가격
```

표시 규칙:

- `100개 이상`: 정수 표시
- `10개 이상 100개 미만`: 소수점 1자리까지 표시
- `1개 이상 10개 미만`: 소수점 1자리까지 표시
- `1개 미만`: 소수점 2자리까지 표시
- 불필요한 `.0`은 제거한다.

예:

- `-50,000원` + `치킨 23,000원` -> `2.2마리`
- `+5,700,000원` + `샤넬백 8,200,000원` -> `0.7개`
- `-200,000,000원` + `포르쉐 자동차 180,000,000원` -> `1.1대`

## 결과 이미지 생성 규칙

- 결과 카드 규격은 `768 x 1152` PNG 기준으로 유지한다.
- 웹 결과 화면과 다운로드 이미지는 같은 계산 결과를 사용한다.
- 배경 이미지는 말풍선 없는 동일 규격 PNG만 사용한다.
- 말풍선 문구가 필요한 경우 텍스트 포함 PNG로 분리한다.
- 결과 텍스트는 HTML/CSS와 canvas 양쪽에서 같은 수량 문자열을 사용한다.
- 아이템 이미지는 중앙 하단에 `contain` 방식으로 배치한다.

## UI 개선 우선순위

1. 금액 입력에 쉼표 허용 및 잘못된 입력 안내
2. 가격대 밖 아이템 제거와 대량 환산 예외 문서화
3. 결과 카드에서 긴 아이템명과 긴 수량 문자열 줄바꿈 안정화
4. 다운로드 이미지와 화면 결과의 위치/크기 일치
5. 모바일 첫 화면에서 입력 카드까지 스크롤 부담 축소
6. 메인 화면 예시 아이템을 실제 가격대 섹션과 연결
7. 공유 버튼 실제 동작 연결
8. 오늘의 현황/희귀 아이템을 mock 데이터 구조로 분리

## 2차 MVP 후보

- 주식 종목/수량/평단 입력
- 국내/미국 종가 기반 평가손익 계산
- 익명 집계 기반 평균 손익/비슷한 사용자 수
- 광고 영역을 고려한 결과 화면 레이아웃
- 데이터 출처 및 투자 조언 아님 고지
